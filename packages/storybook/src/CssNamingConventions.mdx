# CSS Naming Conventions

Consistente naamgeving maakt CSS voorspelbaar: je kunt de naam van een klasse afleiden uit de structuur van een component, en de structuur van een component afleiden uit de naam van een klasse. Dit document legt de regels vast die gelden voor alle CSS in dit design system.

---

## Namespace — `dsn-`

Alle klassen en CSS custom properties beginnen met het prefix `dsn-` (Design System Namespace). Dit voorkomt botsingen met applicatie-CSS of third-party libraries.

```css
/* ✅ correct */
.dsn-button { }
--dsn-color-neutral-bg-default

/* ❌ fout — geen prefix */
.button { }
--color-neutral-bg-default
```

---

## BEM — Block, Element, Modifier

We gebruiken [BEM](https://getbem.com/) als naamgevingsconventie voor CSS-klassen. BEM maakt de relatie tussen HTML-elementen en hun stijlregels expliciet leesbaar zonder de DOM-structuur te kennen.

### Structuur

```
dsn-[block]
dsn-[block]__[element]
dsn-[block]--[modifier]
dsn-[block]__[element]--[modifier]
```

- Enkelvoudige koppeltekens (`-`) binnen een woord: `form-field`, `text-input`
- Dubbele underscore (`__`) voor elementen
- Dubbele koppeltekens (`--`) voor modifiers

### Block

Een block is een zelfstandig, herbruikbaar component. De blocknaam is gelijk aan de componentnaam, in lowercase kebab-case.

```css
.dsn-button { }
.dsn-text-input { }
.dsn-form-field { }
.dsn-status-badge { }
```

### Element

Een element is een onderdeel van een block dat geen zelfstandige betekenis heeft buiten dat block. Elementen worden altijd geschreven als `block__element` — nooit genest (`block__element__sub-element`).

```css
.dsn-button__label { }        /* tekstlabel in een button */
.dsn-alert__heading { }       /* heading in een alert */
.dsn-note__icon { }           /* icoon in een note */
.dsn-text-input__wrapper { }  /* wrapper-div rondom een input */
```

```html
<!-- ✅ Correct — één niveau diep -->
<div class="dsn-alert">
  <span class="dsn-alert__icon">...</span>
  <strong class="dsn-alert__heading">Fout</strong>
  <div class="dsn-alert__content">...</div>
</div>

<!-- ❌ Fout — geneste element-naam -->
<div class="dsn-alert">
  <div class="dsn-alert__content">
    <p class="dsn-alert__content__text">...</p>  <!-- niet doen -->
  </div>
</div>
```

### Modifier

Een modifier past het uiterlijk of gedrag van een block of element aan. Modifiers staan **altijd naast** de basis-klasse, nooit alleen.

```html
<!-- ✅ Correct — modifier naast basis-klasse -->
<button class="dsn-button dsn-button--strong">Label</button>

<!-- ❌ Fout — modifier zonder basis-klasse -->
<button class="dsn-button--strong">Label</button>
```

#### Variant-modifiers

Veranderen het visuele karakter van een component.

```css
.dsn-button--strong { }       /* primaire, prominente knop */
.dsn-button--subtle { }       /* teruggetrokken knop */
.dsn-note--info { }           /* informatieve variant */
.dsn-note--warning { }        /* waarschuwingsvariant */
```

#### Grootte-modifiers

Altijd geschreven als `--size-{naam}`, nooit afgekorte losse woorden.

```css
.dsn-button--size-small { }   /* ✅ */
.dsn-button--small { }        /* ❌ — ontbreekt 'size-' prefix */
.dsn-button--sm { }           /* ❌ — afkorting, niet leesbaar */
```

#### Toestandsmodifiers

Beschrijven een specifieke toestand van een component.

```css
.dsn-button--loading { }         /* laadstatus */
.dsn-button--icon-only { }       /* alleen icoon, label visueel verborgen */
.dsn-note--no-heading { }        /* note zonder heading */
.dsn-checkbox__input--invalid { } /* ongeldig formulierveld */
```

> **Opmerking:** HTML-attributen als `disabled`, `aria-invalid` en `aria-expanded` worden direct als CSS-selector gebruikt voor interactieve toestandsstijlen, niet als BEM-modifier:
>
> ```css
> .dsn-button:disabled { }
> .dsn-text-input:focus { }
> .dsn-text-input[aria-invalid="true"] { }
> ```

## CSS Custom Properties

### Globale design tokens

Globale tokens volgen de structuur `--dsn-{categorie}-{schaal/eigenschap}`. Ze zijn beschikbaar in de volledige applicatie en mogen in elke CSS-context gebruikt worden.

```
--dsn-{categorie}-{subgroep?}-{eigenschap}
```

```css
--dsn-color-neutral-bg-default
--dsn-color-positive-border-active
--dsn-space-block-md
--dsn-border-width-medium
--dsn-focus-outline-color
--dsn-box-shadow-sm
```

### Component tokens

Component tokens zijn beschikbaar op de Design Tokens-pagina en volgen `--dsn-{component}-{variant?}-{eigenschap}`. Ze geven toegang tot component-specifieke waarden die thema's kunnen overschrijven.

```
--dsn-{component}-{variant?}-{eigenschap}
```

```css
--dsn-button-strong-background-color
--dsn-button-strong-color
--dsn-note-info-background-color
--dsn-text-input-focus-border-color
--dsn-alert-padding-block
```

### Lokale scoping — alias-pattern

Binnen een component-CSS-bestand worden lokale custom properties gebruikt als runtime-alias. Dit maakt variant-switching mogelijk zonder specificiteitsproblemen, en houdt de stijlregels voor states compact.

**Patroon:**

```css
/* Stap 1 — definieer de lokale alias op het block, wijs standaardvariant toe */
.dsn-button {
  --dsn-button-background-color: var(--dsn-button-default-background-color);
  --dsn-button-color:            var(--dsn-button-default-color);
}

/* Stap 2 — overschrijf de alias per variant-modifier */
.dsn-button--strong {
  --dsn-button-background-color: var(--dsn-button-strong-background-color);
  --dsn-button-color:            var(--dsn-button-strong-color);
}

/* Stap 3 — gebruik uitsluitend de lokale alias in de stijlregels */
.dsn-button {
  background-color: var(--dsn-button-background-color);
  color:            var(--dsn-button-color);
}

/* Stap 4 — hover/focus hoeft de waarde niet te herhalen, alleen de alias te updaten */
.dsn-button:hover {
  --dsn-button-background-color: var(--dsn-button-default-background-color-hover);
}
```

**Voordelen:**
- Eén set stijlregels voor alle varianten — geen gedupliceerde `background-color`-declaraties per modifier.
- States (`:hover`, `:focus`, `:disabled`) overschrijven enkel de alias, niet de volledige stijlregel.
- Thema-overrides hoeven alleen de component-token aan te passen, niet de component-CSS.

> **Lokale tokens zijn privé.** Ze bestaan alleen binnen de component-CSS en worden nooit extern gedocumenteerd of gebruikt. Alleen de component tokens (in de token-JSON-bestanden) zijn publieke API.

## Uitzondering — utility-klassen

Utility-klassen volgen geen BEM-structuur. Ze zijn globale helpers met één verantwoordelijkheid, en zijn herkenbaar aan hun enkelvoudige naam na het `dsn-`-prefix.

```css
.dsn-visually-hidden { }  /* verberg visueel, maar behoud voor screenreaders */
```

Utility-klassen hebben nooit elementen of modifiers. Ze worden geplaatst op elk element dat de utility nodig heeft, ongeacht het block.

---

## Bestandsstructuur

Elk component-CSS-bestand volgt dezelfde vaste volgorde van secties, gescheiden door een sectionheader-comment:

```css
/* ===========================
   Local color tokens (neutral = default)
   =========================== */

/* Variant-alias overrides */

/* ===========================
   Base layout
   =========================== */

/* ===========================
   [Element A]
   =========================== */

/* ===========================
   [Element B]
   =========================== */

/* ===========================
   [Modifier]
   =========================== */
```

## Snelle referentie

| Situatie | Klasse |
|---|---|
| Zelfstandig component | `dsn-button` |
| Onderdeel van een component | `dsn-button__label` |
| Visuele variant | `dsn-button--strong` |
| Grootte | `dsn-button--size-small` |
| Toestand (visueel) | `dsn-button--loading` |
| Toestand (HTML-attribuut) | `.dsn-button:disabled` |
| Globale token | `--dsn-color-neutral-bg-default` |
| Component token | `--dsn-button-strong-background-color` |
| Lokale alias (interne CSS) | `--dsn-button-background-color` |
| Utility | `dsn-visually-hidden` |

## Veelgemaakte fouten

### Geneste element-namen

```css
/* ❌ — geen geneste elementen */
.dsn-form-field__label__icon { }

/* ✅ — vlak, element van het block */
.dsn-form-field__label-icon { }
```

### Modifier zonder basis-klasse

```html
<!-- ❌ — modifier werkt niet zonder basis-klasse -->
<div class="dsn-note--info">...</div>

<!-- ✅ -->
<div class="dsn-note dsn-note--info">...</div>
```

### Hardcoded waarden in place van tokens

```css
/* ❌ — hardcoded kleur */
.dsn-button--strong {
  background-color: #1b59a4;
}

/* ✅ — via token */
.dsn-button--strong {
  --dsn-button-background-color: var(--dsn-button-strong-background-color);
}
```

### Grootte-modifier zonder `size-` prefix

```css
/* ❌ */
.dsn-button--small { }

/* ✅ */
.dsn-button--size-small { }
```

### Camelcase of PascalCase in klassenamen

```css
/* ❌ */
.dsn-formField { }
.dsn-FormField { }

/* ✅ */
.dsn-form-field { }
```
